\section{Related Work}
The body of related work in this field encompasses three main categories: convergence dynamics in networked systems, the design and function of various routing protocols, and the specific application and characteristics of Switched Ethernet.

\subsection{Convergence Dynamics in Networked Systems}
In the realm of networked systems, particularly during initialization or when undergoing topological alterations, nodes must acquire an understanding of the network graph, which can vary in its degree of completeness. Traditional routing protocols usually achieve this through one of two methods: either by having nodes disseminate their routing tables to others or by initiating a manual topology exploration using flood messages.

A key aspect of the TinyNet protocol is its real-time adaptation to failure scenarios, its agility in rerouting paths in response to link or node failures. This subsection focuses on evaluating TinyNet's proficiency in rapidly reconstituting network connectivity in such situations.

The term ``convergence'' in this context refers to restoring consistent topological knowledge among routers in the network. Within the TinyNet framework, convergence is characterized by the time lapse experienced when a node is eliminated from the network, followed by the ensuing processes of dynamic packet redistribution and rerouting. Note that convergence is slightly different for TinyNet since each node will not have a complete view of the network graph. However, convergence time from other routing protocols will serve as good benchmarks as convergence time measures the time it takes the network to operate normally in the event of a node failure or of node busyness.

\subsection{Design and Function of State-of-the-Art Routing Protocols}
We evaluate the failure response in three prominent protocols: OSPF \cite{ospf}, SPB \cite{spb}, and TRILL \cite{trill}. In these protocols, nodes require complete knowledge of the network graph to determine the shortest path for data transmission. Consequently, a single node failure can pause the entire network, as all nodes must update their network graph representation, leading to a significant reduction in message delivery time predictability. Generally, these protocols are associated with a standard convergence time of approximately 200 milliseconds \cite{eramo2008multi,farkas2009performance}.

TinyNet introduces a more efficient approach for NCS. When a node fails, information about the compromised links and nodes is disseminated throughout the network in a way that maintains the network's current state and minimizes additional delays.

Another critical factor in network resilience is the capacity of routing protocols to utilize multiple paths. OSPF, SPB, and TRILL each incorporate strategies to achieve this. TRILL employs Fabric Shortest Path First (FSPF) to identify alternative routes following topological changes, while OSPF and SPB can utilize equal-cost multipath (ECMP) routing to handle multiple paths of identical cost. However, these protocols typically enable dynamic pathfinding only under specific conditions.

In contrast, TinyNet leverages a more straightforward path selection method, which significantly reduces computational overhead compared to stateful protocols. This is achieved by minimizing packet header sizes and avoiding complex computations, such as running Dijkstra's algorithm on the network graph. As a result, TinyNet is not only more adaptable but also more compatible across a diverse range of hardware environments.

\subsection{Switched Ethernet}
Switched Ethernet operates on the principle of creating a minimum spanning tree, leading to a scenario where nodes within the same layer are in competition for transmission time on the subsequent layer. Consequently, the time delay for message delivery tends to increase in proportion to both the likelihood of simultaneous transmissions by peer nodes and the total number of peers within that layer.

A significant limitation of Switched Ethernet is its vulnerability to single points of failure. In situations where a link or switch fails, the network is compelled to reinitiate the spanning tree protocol algorithm, a process that can extend over several seconds \cite{krishnan2013performance}. Due to the hierarchical nature of Switched Ethernet's network graphs, a failure in a single link can render extensive network sections inaccessible.

Moreover, Switched Ethernet lacks programmability. Its switches are essentially black-box integrated circuits, which restrict the ability of system designers to incorporate additional functionalities directly at the networking layer. This limitation is particularly relevant in NCS, where features such as message prioritization and load balancing are essential. Under Switched Ethernet, these capabilities cannot be integrated into the Layer 2 routing strategy and must instead be implemented at the application layer, leading to potential inefficiencies and complexities in system design.

The IEEE 802.1 Time-Sensitive Networking (TSN) standards \cite{messenger2018tsn} represent a recent effort to add determinism to Ethernet by introducing mechanisms such as credit-based shaping, time-aware scheduling, and frame preemption. While TSN improves delivery determinism within a single network segment, it still relies on centralized network configuration and remains subject to spanning-tree constraints for fault tolerance. TinyNet complements this direction by addressing fault tolerance and multipath routing at the protocol level without requiring centralized management.
